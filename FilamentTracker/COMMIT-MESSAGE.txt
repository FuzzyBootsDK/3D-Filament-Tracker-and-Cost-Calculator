## Title

feat: Add MQTT Relay Server for ESP32 and multi-client support
```

## Description
```
Implements an MQTT relay/rebroadcaster to solve connection stability issues 
when multiple devices try to connect to BambuLab printer's MQTT broker.

### Problem Solved
BambuLab printers have limited concurrent MQTT connection slots (2-5).
When multiple devices (Filament Tracker, ESP32, Home Assistant) connect
directly, connections become unstable and frequently drop.

### Solution
- Filament Tracker maintains single connection to printer
- Built-in MQTT broker rebroadcasts all messages
- Unlimited clients can connect to relay instead of printer
- Improves stability and reduces printer resource usage

### Features Added
- MqttRelayService with full MQTT broker implementation
- Settings UI for relay configuration (port, auth)
- Optional username/password authentication
- Auto-generated ESP32 connection instructions
- Real-time client connection monitoring
- Docker support with port 1883 exposed
- Environment variable configuration
- Comprehensive documentation

### Database Migration
Automatically adds 4 new columns to AppSettings table:
- MqttRelayEnabled (bool, default: false)
- MqttRelayPort (int, default: 1883)
- MqttRelayUsername (string?, nullable)
- MqttRelayPassword (string?, nullable)

### Docker Changes
- Exposes port 1883 for MQTT relay

- Fully documented Docker networking behavior
